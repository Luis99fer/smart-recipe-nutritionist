# Model Pipeline

## Overview

Smart Recipe Nutritionist uses a practical machine learning and recommendation pipeline to evaluate ingredient combinations and suggest recipe-related outputs.

The pipeline combines:

- ingredient parsing;
- feature vectorization;
- machine learning model inference;
- local nutrition lookup;
- optional USDA API fallback;
- cosine similarity recipe search;
- daily menu generation.

The final application is executed through the command line.

---

## High-Level Pipeline

```text
User ingredients
        ↓
Ingredient parsing and normalization
        ↓
Binary feature vector creation
        ↓
ML model prediction
        ↓
Nutrition lookup
        ↓
Similar recipe search
        ↓
Formatted CLI output
```

For daily menu generation:

```text
Processed recipe dataset
        ↓
Meal type inference
        ↓
Recipe nutrition lookup
        ↓
Recipe scoring
        ↓
Breakfast, lunch, dinner selection
        ↓
Formatted daily menu
```

---

## 1. User Input

The user provides ingredients through the command line.

Example:

```bash
python src/nutritionist.py chicken, tomato, garlic
```

The app also supports:

```bash
python src/nutritionist.py "chicken, tomato, garlic"
```

The daily menu mode is executed with:

```bash
python src/nutritionist.py --menu
```

---

## 2. Ingredient Parsing

Ingredient parsing is handled by:

```text
NutritionistApp.parse_cli_ingredients()
```

The method:

- accepts multiple command-line arguments;
- splits comma-separated values;
- normalizes text to lowercase;
- removes empty values;
- removes duplicates while preserving order.

Example input:

```text
Chicken, tomato, garlic, chicken
```

Parsed output:

```python
["chicken", "tomato", "garlic"]
```

---

## 3. Feature Vectorization

The ML model requires input ingredients to be represented as a binary feature vector.

This is handled by:

```text
RatingForecaster._vectorize()
```

The feature column order is loaded from:

```text
models/feature_columns.joblib
```

For each feature column:

```text
1 = ingredient is present
0 = ingredient is absent
```

Example:

```text
Input ingredients:
chicken, tomato, garlic
```

Possible vector representation:

```text
chicken = 1
tomato = 1
garlic = 1
beef = 0
onion = 0
...
```

The final vector is passed to the trained model as a one-row pandas DataFrame.

---

## 4. ML Forecast

The model is loaded by:

```text
RatingForecaster
```

Model file:

```text
models/best_rating_label_model.joblib
```

Feature columns file:

```text
models/feature_columns.joblib
```

The prediction method is:

```python
forecaster.predict_label(ingredients)
```

The model predicts a label such as:

```text
bad
so-so
great
```

The label is converted into a user-facing message by:

```text
RatingForecaster.label_to_message()
```

Example output:

```text
I. OUR FORECAST
This looks like a great combination. We think it has a strong chance to become a tasty dish.
```

## 5. Nutrition Lookup

Nutrition facts are handled by:

```text
NutritionLookup
```

The application first checks the local processed nutrition file:

```text
data/ingredient_nutrition_daily_values.csv
```

This file contains nutrition values expressed as percentages of recommended daily values.

Example nutrients:

```text
protein
total carbohydrate
total fat
saturated fat
cholesterol
sodium
fiber
sugars
calcium
iron
potassium
vitamin d
magnesium
zinc
```

The nutrition section is generated by:

```text
NutritionistApp.nutrition_section()
```

Example output:

```text
II. NUTRITION FACTS
Chicken
Protein - 62% of Daily Value
Sodium - 4% of Daily Value
```

## 6. USDA API Fallback

If an ingredient is not found in the local nutrition CSV, the app can optionally use the USDA FoodData Central API.

The API key is read from the environment variable:

```text
USDA_API_KEY
```

It should not be hardcoded in the source code.

Windows PowerShell:

```powershell
$env:USDA_API_KEY="your_api_key_here"
```

macOS/Linux:

```bash
export USDA_API_KEY="your_api_key_here"
```

If no API key is provided, the app still works using the local nutrition CSV.

The fallback request is made by:

```text
NutritionLookup.fetch_ingredient()
```

---

## 7. Similar Recipe Search

Similar recipe recommendation is handled by:

```text
SimilarRecipesEngine
```

The processed recipe data is loaded from:

```text
data/similar_recipes.csv
```

This file contains:

- recipe title;
- recipe rating;
- recipe URL;
- binary ingredient columns.

The engine creates:

```text
1. A binary ingredient matrix for all recipes
2. A binary query vector from user ingredients
3. Cosine similarity scores between the query and all recipes
```

The similarity method is:

```text
cosine similarity
```

The output is the top-k most similar recipes.

Default:

```text
top 3 recipes
```

Example output:

```text
III. TOP-3 SIMILAR RECIPES:
- Chicken Tomato Pasta, rating: 4.3, URL:
https://example.com/recipe
```

## 9. Meal Type Inference

Meal type is inferred using keyword rules.

Breakfast keywords include:

```text
breakfast
omelet
omelette
muffin
pancake
granola
oatmeal
toast
egg
```

Lunch keywords include:

```text
salad
sandwich
wrap
soup
bowl
```

Dinner keywords include:

```text
steak
chicken
beef
pasta
roast
dinner
salmon
pork
```

If no keyword is matched, the recipe defaults to:

```text
LUNCH
```

---

## 10. Recipe Scoring

Recipe scoring is implemented in:

```text
DailyMenuGenerator.recipe_score()
```

The score combines:

- recipe rating;
- total nutrition contribution;
- penalty for excessive nutrient values.

General scoring idea:

```text
score = rating contribution + nutrition contribution - excess nutrient penalty
```

This is a heuristic score, not a medical recommendation.

```text
NutritionistApp.run()
```

For ingredient mode, the output contains:

```text

II. NUTRITION FACTS
```

For daily menu mode, the output contains:

```text
BREAKFAST
```

Each meal includes:

- recipe title;
- rating;
- ingredients;
- nutrition values;
- URL.

---

## 12. Main Classes in the Pipeline

| Class | Responsibility |
|---|---|
| `RecipesData` | Loads recipe datasets and detects ingredient columns. |
| `NutritionLookup` | Retrieves nutrition facts from local CSV or USDA API. |
| `SimilarRecipesEngine` | Finds similar recipes using cosine similarity. |
| `RatingForecaster` | Loads the trained model and predicts ingredient combination quality. |
| `DailyMenuGenerator` | Generates breakfast, lunch, and dinner recommendations. |
| `NutritionistApp` | Connects all components into one CLI application. |

---

## 13. Required Runtime Files

To run the full app, the following files are required:

```text
data/ingredient_nutrition_daily_values.csv
data/similar_recipes.csv
models/best_rating_label_model.joblib
models/feature_columns.joblib
```

The raw dataset is not required for normal app usage.
