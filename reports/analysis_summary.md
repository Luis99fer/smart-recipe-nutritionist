# Analysis Summary

## Project

**Smart Recipe Nutritionist**

Smart Recipe Nutritionist is a command-line Python application that combines machine learning, nutrition lookup, and recipe similarity search.

The project receives a list of ingredients and returns:

```text
I. OUR FORECAST
II. NUTRITION FACTS
III. TOP-3 SIMILAR RECIPES
```

It can also generate a simple daily menu with:

```text
BREAKFAST
LUNCH
DINNER
```

---

## 1. Application Workflow

The application workflow is organized as follows:

```text
User ingredients
        ↓
Ingredient parsing
        ↓
Feature vectorization
        ↓
ML label prediction
        ↓
Nutrition lookup
        ↓
Similar recipe search
        ↓
Formatted CLI output
```

Daily menu mode follows a second workflow:

```text
Processed recipe dataset
        ↓
Meal type inference
        ↓
Recipe scoring
        ↓
Breakfast / lunch / dinner selection
        ↓
Formatted daily menu
```

---

## 2. Ingredient Forecast

The ingredient forecast is implemented through the `RatingForecaster` class.

The model loads two files:

```text
models/best_rating_label_model.joblib
models/feature_columns.joblib
```

The user-provided ingredients are converted into a binary feature vector.

Example input:

```text
chicken, tomato, garlic
```

Example vector logic:

```text
chicken = 1
tomato = 1
garlic = 1
beef = 0
onion = 0
...
```

The model predicts one of the following labels:

```text
bad
so-so
great
```

The predicted label is converted into a readable forecast message.

---

## 3. Nutrition Facts

Nutrition facts are handled by the `NutritionLookup` class.

The application first checks the local processed nutrition file:

```text
data/ingredient_nutrition_daily_values.csv
```

This file stores ingredient-level nutrition data as percentages of recommended daily values.

Possible nutrient fields include:

```text
protein
total carbohydrate
total fat
saturated fat
cholesterol
sodium
fiber
sugars
calcium
iron
potassium
vitamin d
magnesium
zinc
```

If an ingredient is not found locally, the application can optionally use the USDA FoodData Central API.

The API key is read from:

```text
USDA_API_KEY
```

and should never be hardcoded in the source code.

---

## 4. Similar Recipe Search

Similar recipes are generated by the `SimilarRecipesEngine` class.

The app uses:

```text
data/similar_recipes.csv
```

This file contains recipe titles, ratings, URLs, and binary ingredient columns.

The recommendation process uses cosine similarity between:

```text
user ingredient vector
```

and

```text
recipe ingredient vectors
```

The app returns the top 3 similar recipes by default.

Example output structure:

```text
III. TOP-3 SIMILAR RECIPES:
- Recipe Title, rating: 4.5, URL:
https://example.com/recipe
```

---

## 5. Daily Menu Generation

Daily menu generation is handled by the `DailyMenuGenerator` class.

The application can generate:

```text
BREAKFAST
LUNCH
DINNER
```

The generator uses:

- recipe title keywords;
- recipe ratings;
- available nutrition facts;
- a simple nutrition-based scoring function.

Meal type inference is keyword-based.

Examples:

```text
omelet, pancake, oatmeal → BREAKFAST
salad, sandwich, soup → LUNCH
chicken, beef, pasta, salmon → DINNER
```

If no clear keyword is found, the recipe defaults to:

```text
LUNCH
```

## 6. Data and Model Artifacts

The project requires the following processed data files:

```text
data/ingredient_nutrition_daily_values.csv
data/similar_recipes.csv
```

The project also requires the following model files:

```text
models/best_rating_label_model.joblib
models/feature_columns.joblib
```

These files are included so that the final application can run directly after installing dependencies.

The raw development dataset:

```text
data/epi_r.csv
```

is not required for normal application usage and is not included in GitHub.

## 7. Main Python Components

The main source files are:

```text
src/recipes.py
src/nutritionist.py
```

### `src/recipes.py`

Contains the core application logic:

| Class | Responsibility |
|---|---|
| `RecipesData` | Loads recipe datasets and detects ingredient columns. |
| `NutritionLookup` | Retrieves nutrition facts from local CSV or USDA API. |
| `SimilarRecipesEngine` | Finds similar recipes using cosine similarity. |
| `RatingForecaster` | Loads the trained model and predicts ingredient combination quality. |
| `DailyMenuGenerator` | Generates a daily menu. |
| `NutritionistApp` | Connects all components into a single application interface. |

### `src/nutritionist.py`

Contains the command-line entry point.

Example commands:

```bash
python src/nutritionist.py chicken, tomato, garlic
python src/nutritionist.py --menu
```
